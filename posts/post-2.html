在 GitHub Pages 上部署静态站点

2025-02-03

GitHub Pages 允许你从仓库的某个分支或目录直接发布静态网页，无需自备服务器。

两种常用方式

方式一：从分支发布。在仓库 Settings → Pages 里，Source 选择分支（如 main），再选择根目录或 /docs，保存后等待几分钟，站点会出现在 https://用户名.github.io/仓库名/。

方式二：用 GitHub Actions 部署。工作流在每次 push 时把构建产物推送到 gh-pages 分支，Pages 从该分支发布。适合需要先构建（如 Jekyll、Hugo）或希望源码与输出分离的情况。

自定义域名

在 Pages 设置里填写自定义域名，并在 DNS 中添加 CNAME 或 A 记录指向 GitHub，即可用自有域名访问。
